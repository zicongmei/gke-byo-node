# NixOS GKE Node Setup Script (`setup_nix_node.py`)

This script automates the process of registering a NixOS machine as an unmanaged worker node in a GKE cluster. It bridges the gap between Google's standard Kubernetes binaries and the declarative, non-FHS nature of NixOS.

## Technical Overview

The script performs four primary phases to integrate the machine into the cluster:

### 1. Credential Placement
It decodes base64-encoded credentials (CA cert, node keys, and client certs) provided by `generate_node_args.py` and places them in standard locations:
- `/etc/kubernetes/pki/ca.crt`
- `/var/lib/kubelet/node.crt` and `node.key`
- `/etc/kubernetes/local-edit.crt` and `local-edit.key` (for administrative tasks)

### 2. Arbitrary Version Support & Binary Patching
Unlike standard Linux distributions, NixOS cannot run raw binaries from `dl.k8s.io` because it lacks a standard `/lib64/ld-linux-x86-64.so.2` path. 

To support any version (e.g., v1.35.2):
- **Download**: It fetches `kubelet`, `kubectl`, and `kube-proxy` from `dl.k8s.io`.
- **Nix Derivation**: It generates a `stdenv.mkDerivation` that uses `pkgs.autoPatchelfHook`. This Nix tool identifies the binaries' library dependencies and updates their ELF headers to point to the correct paths in the `/nix/store`.
- **Kubelet Wrapper**: It implements a shell wrapper for `kubelet` that filters out the `--pod-infra-container-image` flag. This is necessary because newer Kubernetes versions (1.31+) have removed this flag, but the NixOS module system still attempts to pass it.

### 3. Declarative Configuration Generation
The script generates a dedicated Nix file at `/etc/nixos/kubernetes-node.nix`. This file uses the `services.kubernetes` module but overrides the `package` attribute with the custom-patched binaries created in Step 2.

Key configurations include:
- **Containerd**: Enabled with `SystemdCgroup = true`.
- **CNI**: Configures a bridge network (`cni0`) using the node's assigned `podCIDR`.
- **Kubelet**: Configured with the correct API server URL, DNS IP, and mandatory GKE labels.

### 4. System Activation
Finally, the script ensures `/etc/nixos/kubernetes-node.nix` is imported into the main system configuration and runs `nixos-rebuild switch`. This triggers Nix to:
1. Build the custom patched Kubernetes package.
2. Download all required dependencies (like `socat`, `conntrack`, `iptables`).
3. Generate and start the `kubelet.service` and `kube-proxy.service` systemd units.

## Usage

This script is intended to be run **on the target NixOS VM** as root, using the command generated by `generate_node_args.py`.

```bash
sudo python3 setup_nix_node.py \
  --name "my-nixos-node" \
  --api-url "https://<API_IP>" \
  --version "1.35.2" \
  --ca-cert-base64 "..." \
  [other credentials]
```

## Requirements
- NixOS 24.11 or newer (for proper list-based `clusterDns` support).
- Python 3.x (available in the base NixOS GCP image).
- Internet access (to fetch binaries and Nix packages).
